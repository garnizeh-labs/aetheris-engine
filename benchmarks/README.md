# Aetheris Benchmarks

This directory contains the historical record of performance benchmarks for the Aetheris Engine. The goal is to maintain an auditable trail of performance deltas and memory allocation patterns throughout the development lifecycle.

## Structure
- `_template/`: Standard template for performance analysis reports.
- `<YYYY-MM-DD_HHMM>/`: Dated directories containing:
  - `criterion/`: Raw data and HTML reports generated by Criterion.
  - `REPORT.md`: Detailed analysis of the results and comparison with previous baselines.

## How to Record a New Benchmark
To run the benchmarks and automatically persist the results in history, use the following command:

```bash
just bench-record
```

This command will:
1. Execute `cargo bench --workspace`.
2. Create a timestamped subfolder in `benchmarks/`.
3. Copy the artifacts from `target/criterion` to the historical record folder.

## How to Compare
To compare your current state with a specific baseline saved in history:

1. Identify the reference benchmark folder (e.g., `benchmarks/2026-04-23_1038/`).
2. Use the HTML report link inside `criterion/report/index.html` for graphical visualization.
3. Consult the `REPORT.md` within that folder to understand the context of that measurement.

## Retention Policy
- **Milestone Baselines**: Benchmarks associated with a major vertical slice (VS-XX) or release are kept indefinitely.
- **Daily Runs**: Automated or manual daily runs are kept for 30 days before being purged.
- **Raw Data**: The `criterion/` folder is ignored by git to avoid bloat. Only `REPORT.md` is committed to history. Full artifacts should be stored in external storage if needed for deep analysis.
